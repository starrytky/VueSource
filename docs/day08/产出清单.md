# Day08 产出清单

> 根据 `VueSource/docs/vue源码阅读第八天打断点.md` 整理  
> 目标：把“第八天到底要产出什么”收敛成可执行、可验收的清单

## 1. Day08 核心目标

第八天的目标不是继续停留在模板编译，而是把 `.vue` 单文件组件的编译主链亲手走通。

第八天只要求走通下面这条主链：

`.vue source`  
`-> parse(...)`  
`-> descriptor`  
`-> compileScript(...)`  
`-> compileTemplate(...)`  
`-> compileStyle(...)`  
`-> 重新拼装组件模块`

只要这条链路能通过断点走通，并且能用自己的话讲清楚，Day08 就算完成。

## 2. 必须产出

### 2.1 一份 SFC 编译总链笔记

至少要整理出下面这条调用链：

```text
.vue source
  -> parse
  -> descriptor
  -> compileScript
  -> compileTemplate
  -> compileStyle
  -> final module
```

要求：

- 说明每一步在哪个文件
- 说明 descriptor 为什么是第一步关键产物

### 2.2 一份 descriptor 结构总结

至少要说明：

- `template`
- `script`
- `scriptSetup`
- `styles`
- `customBlocks`

要求：

- 能解释为什么 `.vue` 文件必须先拆成结构化描述对象

### 2.3 一份 `script setup` 宏总结

至少要覆盖：

- `defineProps`
- `defineEmits`
- `defineExpose`
- `defineModel`

要求：

- 能解释为什么这些 API 本质上是编译期宏
- 能说明它们大概在 `compiler-sfc/src/script/*` 哪些文件里落地

### 2.4 一份 template / style 接回总结

至少要说明：

1. `compileTemplate` 会接回 `compiler-dom` 或 SSR compiler
2. `compileStyle` 负责 scoped style / css vars 这类样式编译工作
3. SFC id 为什么要贯穿脚本、模板、样式几部分

### 2.5 一份这 5 个问题的答案

至少能回答：

1. `.vue` 文件怎样先变成 descriptor？
2. `compileScript` 怎样统一普通 script 和 script setup？
3. 为什么 `defineProps / defineEmits / defineModel` 属于编译期宏？
4. `compileTemplate` 怎样接回 `compiler-dom`？
5. scoped style 大概在哪个阶段处理？

## 3. 建议产出形式
