# Day06 产出清单

> 根据 `VueSource/docs/vue源码阅读第六天打断点.md` 整理  
> 目标：把“第六天到底要产出什么”收敛成可执行、可验收的清单

## 1. Day06 核心目标

第六天的目标不是继续停留在 runtime，而是把 `compiler-core` 的三段式主链亲手走通。

第六天只要求走通下面这条主链：

`baseCompile(template)`  
`-> baseParse(...)`  
`-> transform(...)`  
`-> generate(...)`  
`-> render code`

只要这条链路能通过断点走通，并且能用自己的话讲清楚，Day06 就算完成。

## 2. 必须产出

### 2.1 一份编译三阶段主链笔记

至少要整理出下面这条调用链：

```text
template
  -> baseParse
  -> transform
  -> generate
  -> render code
```

要求：

- 说明每一步在哪个文件
- 说明每一步的职责是什么
- 说明为什么 transform 是最关键的一层

### 2.2 一份 AST 认知总结

至少要说明：

1. AST 是模板的中间表示
2. parse 负责把字符串还原成 AST
3. transform 负责在 AST 上做分析和重写
4. codegen 负责把处理后的 AST 输出成代码

### 2.3 一份核心 transform 对照表

至少要覆盖：

- `transformElement`
- `transformText`
- `transformIf`
- `transformFor`

要求：

- 每个 transform 都要说明“它解决什么问题”
- 最好能配一句对应的模板语义说明

### 2.4 一份这 5 个问题的答案

至少能回答：

1. `baseCompile` 怎样串起 parse、transform、generate？
2. AST 在 Vue 编译器里承担什么角色？
3. 为什么 transform 是最关键的一层？
4. 这些核心 transform 分别在做什么？
5. codegen 为什么输出 helper 调用而不是直接 DOM 指令？

### 2.7 一份验收问答

至少要能独立回答：

1. 模板怎样从字符串变成 render code？
2. 为什么 AST 对编译器是必要的？
3. transform 为什么比 parse / codegen 更关键？
4. 为什么 render code 最终是 helper 调用？
5. 编译器优化信息大致在哪个阶段开始出现？
