# Day09 产出清单

> 根据 `VueSource/docs/vue源码阅读第九天打断点.md` 整理  
> 目标：把“第九天到底要产出什么”收敛成可执行、可验收的清单

## 1. Day09 核心目标

第九天的目标是给这一轮主线收尾：

- 上午走通 SSR / hydration 主链
- 下午把当前仓库 `3.5.30` 的重点新能力做源码落点整理

第九天只要求走通下面两条主链：

### SSR / hydration

`renderToString(...)`  
`-> renderComponentVNode(...)`  
`-> renderVNode(...)`

`hydrate(...)`  
`-> hydrateNode(...)`  
`-> hydrateElement / hydrateChildren / hydrateFragment`

### Vue 3.5 专题落点

- `onWatcherCleanup`
- `pause / resume`
- `defineModel`
- `useTemplateRef`
- lazy hydration strategies

只要这两部分都能通过断点或源码定位走通，并且能用自己的话讲清楚，Day09 就算完成。

## 2. 必须产出

### 2.1 一份 SSR / CSR / hydration 对照笔记

至少要说明：

- SSR 的目标是输出 HTML
- CSR mount 的目标是创建 DOM
- hydration 的目标是接管已有 DOM

建议最少整理出这张表：

| 主题 | 核心目标 | 关键入口 |
| --- | --- | --- |
| SSR | 输出 HTML | `renderToString` |
| CSR mount | 创建 DOM | `render` / `patch` |
| hydration | 接管已有 DOM | `hydrate` / `hydrateNode` |

### 2.2 一份 hydration 主链笔记

至少要整理出下面这条调用链：

```text
hydrate
  -> hydrateNode
  -> hydrateElement / hydrateChildren / hydrateFragment
```

要求：

- 能解释为什么 hydration 不是重新 mount
- 能解释 mismatch 大概会在这条链哪里被发现

### 2.3 一份 Vue 3.5 主题落点表

至少要整理出下面这些主题：

- `onWatcherCleanup`
- `pause / resume`
- `defineModel`
- `useTemplateRef`
- lazy hydration strategies

建议最少整理成下面这种粒度：

| 主题 | 源码位置 | 属于哪层 |
| --- | --- | --- |
| `onWatcherCleanup` | `reactivity/src/watch.ts` | reactivity |
| `pause / resume` | `reactivity/src/effect.ts` | reactivity |
| `defineModel` | `runtime-core` + `compiler-sfc` | runtime + compiler |
| `useTemplateRef` | `runtime-core/src/helpers/useTemplateRef.ts` | runtime-core |
| lazy hydration | `runtime-core/src/hydrationStrategies.ts` | runtime-core |

### 2.4 一份这 5 个问题的答案

1. 服务端渲染时，Vue 是怎样把组件树变成 HTML 的？

3. `hydrateNode / hydrateElement / hydrateChildren` 这条链在做什么？
4. lazy hydration strategy 为什么属于 Vue 3.5 值得单独看的能力？
5. 这些 3.5 能力分别落在哪些模块？
